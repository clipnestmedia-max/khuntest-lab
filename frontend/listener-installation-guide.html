KhunTest Lab Listener Installation Guide

Use this guide on the Windows computer connected to the laboratory analyzer. Run a dummy analyzer test before processing real patient data.

Back to Admin Dashboard Release Notes

Install Or Update

  1. Download KhunTestLab-Listener-Setup.exe from Admin Dashboard > Machine Results.
  2. Close the currently running KhunTest Lab Listener from the system tray before installing an update.
  3. Run the installer. Use administrator mode only if Windows requires it for the selected install location.
  4. Select the preferred installation option and finish setup.
  5. When Windows Firewall prompts for network access, allow Private Network access for the listener.
  6. Open KhunTest Lab Listener and login with an authorized KhunTest Lab admin or lab account.
  7. Open Analyzer Settings and select Mindray BC5000.
  8. Select TCP Server mode.
  9. Set Local Listener Port to 5001.
  10. Start Listener.
  11. Confirm Firebase Connected, Internet Connected, Listener Running, and Waiting for Analyzer or Analyzer Connected.
  12. Configure the analyzer LIS destination IP to the customer PC Ethernet IP and port 5001.
  13. Run a dummy test before using real patient data.

Troubleshooting

  • ECONNREFUSED: The listener is likely in TCP Client mode or connecting to the wrong analyzer endpoint. For Mindray BC5000 inbound LIS mode, use TCP Server mode and port 5001.
  • Port already in use: Close duplicate listener instances, then check netstat -ano | findstr :5001 and stop the conflicting process if it is not the KhunTest listener.
  • Firewall blocked: Add an inbound Windows Firewall rule for TCP port 5001 on Private networks. Do not disable Windows Firewall.
  • Analyzer not connected: Confirm the analyzer LIS destination IP is the customer PC Ethernet IP, for example 10.0.0.3, and that both devices are on the same analyzer network.
  • No raw message received: Check the listener diagnostics. Remote Connections should increase when the analyzer connects, and Last Received Byte should change when it sends data.
  • Firebase upload failed: Confirm the app is logged in with an authorized admin/lab account, Internet Connected is Yes, and Firebase Connected is Yes. Failed uploads should enter the offline queue.
  • Installer cannot replace running application: Exit KhunTest Lab Listener from the system tray, wait a few seconds, then run the installer again.

Security Notes

Windows may show a SmartScreen warning because the installer is not yet code-signed. Verify the publisher, release page, and SHA-256 checksum before installation.

Production distribution should use a code-signing certificate so Windows can verify the installer publisher.
